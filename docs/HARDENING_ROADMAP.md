## Enterprise Tier

Features required before Spectra AI could be offered as a multi-tenant SaaS product or deployed inside an enterprise. None of these are in scope for the current build — they are captured here for completeness.

### Authentication & Identity

- **Google / GitHub SSO** — OAuth 2.0 login via social providers. Removes password management and aligns with how most engineering teams already authenticate internal tools.
- **SAML / OIDC enterprise IdP** — connect to Okta, Azure AD, or any enterprise identity provider. Required for corporate procurement and single sign-on policies.
- **Multi-factor authentication (MFA)** — TOTP or hardware key enforcement for admin accounts. Currently the demo account has no second factor.

### Multi-tenancy & Access Control

- **Organisation workspaces** — each customer org gets an isolated workspace with its own jobs, results, and governance traces. Currently the schema has RLS by `user_id` but there is no concept of an org or team.
- **Role-based access (admin / analyst / viewer)** — admins manage settings and quotas; analysts submit jobs and view results; viewers see reports only. Currently all authenticated users have the same permissions.
- **Per-org modality and model settings** — each org configures which modalities are active and may bring their own API keys for Anthropic or OpenAI. Currently model routing is hardcoded in the agent graph constants.

### Notifications & Integrations

- **Email / Slack on job completion** — notify the team when a pipeline run finishes or fails. Currently users must poll the dashboard UI.
- **Webhook on job failure or low-confidence result** — fire a POST to a configured URL when a job fails or the auditor scores overall faithfulness below a configurable threshold. Allows integration with PagerDuty, OpsGenie, or a custom alerting stack.
- **Programmatic API access** — issue per-org API keys so external systems can submit jobs and retrieve results without using the web UI, closing the automation loop for document-heavy workflows.

### Billing & Usage

- **Per-org API cost tracking** — Lambda already logs per-job model costs to CloudWatch; surfacing this as a monthly aggregate per org with budget alerts is the next step.
- **Tiered rate limits** — current cap (3 jobs/day/IP) is fixed. Enterprise plans would need configurable limits per org and per user tied to a billing tier.
- **Usage dashboard** — show jobs consumed, model tokens used (by modality), and vector index size per org per billing period.

### Audit & Compliance

- **Exportable audit log** — full record of who submitted which job, when, what files were uploaded, and what the governance trace produced — exportable as CSV or JSON for compliance review.
- **Data retention policies** — configurable retention windows per org (e.g. delete jobs and vectors older than 90 days). Currently data is retained indefinitely and Upstash Vector cleanup only runs per-job.
- **SOC 2 / GDPR alignment** — data residency controls, right-to-erasure flows, and a formal security review. AWS `eu-west-1` (Ireland) covers EU data residency for Lambda and S3; Supabase region selection and a formal DPA are needed for the rest.

## Future Modalities

Architectural extensions beyond the current three-modality pipeline (document, vision, audio). None are in scope for the current build — captured here for completeness and interview preparation.

| Item | Why | Skill | Effort |
| :--- | :--- | :--- | :--- |
| **Video (Fourth Modality)** | Video bundles vision and audio into one node: a keyframe sampler feeds GPT-4o (same as `visionNode`) and the extracted audio track feeds Whisper → Claude Sonnet (same as `audioNode`). Guardrail stack: duration/size cap at ingest, EXIF/GPS metadata strip, injection gate + PII redaction on audio transcript, PII redaction on GPT-4o frame descriptions. **New hard problem:** faces in frames are biometric PII — requires AWS Rekognition face detection + blur before GPT-4o receives the frame. Text-level `redactPii` covers OCR-visible PII (emails, SSNs visible in video frames) but cannot redact faces. Cost: a video job invokes GPT-4o (frames) + Whisper + Claude Sonnet — equivalent to ~3 standard jobs; rate limiter weight should reflect this. See [TECHNICAL_ADVISORY.md §25](./TECHNICAL_ADVISORY.md#25-video-as-fourth-modality--architecture-and-guardrail-considerations) for the full architecture and guardrail breakdown. | 2 · Tool & Contract Design | High |
